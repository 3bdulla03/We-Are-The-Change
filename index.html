Introduction

"We Are The Change" is a secure mobile civic engagement platform that enables Bahraini residents to report local community issues and participate in volunteer activities. The system is built with FlutterFlow for the frontend and Firebase for authentication and data management.

Abstract

This project presents the design and development of "We Are The Change," a civic engagement platform that enables citizens to report local issues and participate in community volunteering through a mobile application. Addressing the problem of inefficient public issue reporting and limited citizen engagement, the system allows users to submit reports with photos and sign up for volunteer activities managed by administrators. The project involved system analysis, security requirements engineering, application of threat modeling methodologies, and the implementation of secure user authentication and data handling with Firebase. Testing demonstrated reliable performance, robust data protection, and smooth user experience across devices. Results show the platform effectively streamlines issue reporting, supports secure data management, and encourages greater community participation, confirming the solution's practical value and scalability.

Problem Statement

  • Reporting via email, WhatsApp, and Tawasul is fragmented and slow.
  • No structured coordination between reported needs, volunteers, and sponsors.
  • Security and privacy risks in user data, media uploads, and platform APIs.
  • Limited follow‑up on municipal issues reduces trust and civic engagement.

System Features

  • Issue reporting with photos, location, and contact details.
  • Volunteer browsing and one‑click sign‑up for cases and events.
  • Sponsor browsing and support expression for selected cases.
  • Admin dashboard to create, edit, and manage cases, events, and requests.

Security & Methodology

  • Firebase Authentication with role‑based access control and OTP.
  • Security requirements engineered using the SQUARE methodology.
  • Threat modeling using STRIDE to identify and mitigate risks.
  • Input validation, secure media handling, and restrictive database rules.

System Screens

Technology Stack

  • Frontend: FlutterFlow mobile UI builder.
  • Backend: Firebase Authentication, Firestore, and Storage.
  • Security Methods: SQUARE (requirements), STRIDE (threat modeling).
  • Other: Role‑based access control and OTP verification.

Results

  • Working cross‑platform prototype for civic issue reporting and volunteering.
  • Secure authentication flow with Firebase and OTP‑based verification.
  • End‑to‑end workflows for reporting, volunteering, and sponsor support.
  • Key threats identified and mitigated using STRIDE‑based analysis.

Future Work

  • Municipal Integration: API integration with Bahrain municipalities for automated issue routing and status updates.
  • AI-Powered Triage: Machine learning classification of reported issues for priority assignment and resource allocation.
  • Real-time Notifications: Push notifications for case updates, volunteer assignments, and community events.
  • Analytics Dashboard: Advanced analytics for impact measurement, volunteer engagement metrics, and issue resolution trends.
  • Multi-language Support: Arabic/English localization and right-to-left layout support for broader accessibility.
  • Gamification: Points, badges, and leaderboards to encourage sustained volunteer participation.
